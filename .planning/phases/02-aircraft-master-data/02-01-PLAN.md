    1. `aircraft` table: id SERIAL PK, msn INTEGER NOT NULL UNIQUE, aircraft_type TEXT NOT NULL DEFAULT 'A320', registration TEXT, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ. Index on msn.

    2. `aircraft_rates` table: id SERIAL PK, aircraft_id INTEGER NOT NULL UNIQUE REFERENCES aircraft(id) ON DELETE CASCADE. Fixed monthly rates (USD): lease_rent_usd NUMERIC(12,2), six_year_check_usd NUMERIC(12,2), twelve_year_check_usd NUMERIC(12,2), ldg_usd NUMERIC(12,2). Variable rates per engine (USD): apu_rate_usd NUMERIC(10,4), llp1_rate_usd NUMERIC(10,4), llp2_rate_usd NUMERIC(10,4). Escalation rates: epr_escalation NUMERIC(6,4) DEFAULT 0, llp_escalation NUMERIC(6,4) DEFAULT 0, af_apu_escalation NUMERIC(6,4) DEFAULT 0. Timestamps.

    3. `epr_matrix_rows` table: id SERIAL PK, aircraft_id INTEGER NOT NULL REFERENCES aircraft(id) ON DELETE CASCADE, cycle_ratio NUMERIC(6,4) NOT NULL, benign_rate NUMERIC(10,2) NOT NULL, hot_rate NUMERIC(10,2) NOT NULL, created_at TIMESTAMPTZ. UNIQUE(aircraft_id, cycle_ratio). Index on (aircraft_id, cycle_ratio).

    CRITICAL: All NUMERIC columns must have explicit precision and scale to avoid asyncpg scientific notation serialization (Pitfall 1 from research).

    Create `fastapi-project/app/aircraft/schemas.py` with Pydantic models following the existing `app/users/schemas.py` pattern:
    - `AircraftListResponse`: id, msn, aircraft_type, registration, lease_rent_usd (Decimal|None), six_year_check_usd, twelve_year_check_usd, ldg_usd. These are the list-view fields.
    - `EprMatrixRow`: cycle_ratio (Decimal), benign_rate (Decimal), hot_rate (Decimal)
    - `AircraftDetailResponse`: All fields from list response PLUS apu_rate_usd, llp1_rate_usd, llp2_rate_usd, epr_escalation, llp_escalation, af_apu_escalation (all Decimal|None), and epr_matrix (list[EprMatrixRow])

    - Use `from __future__ import annotations` for Python 3.9 compatibility

    Create `fastapi-project/app/aircraft/repository.py` extending BaseRepository:
    - `list_aircraft(search: str | None = None)` — SELECT aircraft joined with aircraft_rates. If search provided, WHERE msn::TEXT ILIKE or registration ILIKE. ORDER BY msn.
    - `fetch_by_msn(msn: int)` — SELECT aircraft.* joined with all rate fields WHERE msn = $1
    - `fetch_by_id(aircraft_id: int)` — SELECT aircraft WHERE id = $1
    - `fetch_epr_matrix(aircraft_id: int)` — SELECT cycle_ratio, benign_rate, hot_rate FROM epr_matrix_rows WHERE aircraft_id = $1 ORDER BY cycle_ratio
    - `update_rates(aircraft_id: int, **fields)` — Dynamic SET clause (same pattern as UserRepository.update_user). Returns updated row.
    - `create_aircraft(msn: int, aircraft_type: str, registration: str | None)` — INSERT RETURNING *
    - `upsert_rates(aircraft_id: int, ...)` — INSERT ON CONFLICT (aircraft_id) DO UPDATE for all rate fields
    - `upsert_epr_row(aircraft_id: int, cycle_ratio, benign_rate, hot_rate)` — INSERT ON CONFLICT (aircraft_id, cycle_ratio) DO UPDATE
    - Use `from __future__ import annotations` for Python 3.9 compatibility
